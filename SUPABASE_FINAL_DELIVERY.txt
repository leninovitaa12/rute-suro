7. HOW BACKEND USES THESE QUERIES
================================================================================

Backend FastAPI (/backend/main.py) menggunakan queries ini untuk endpoints:

GET /events
  └─ SELECT * FROM events_active ORDER BY start_time;

GET /events/<id>
  └─ SELECT * FROM events WHERE id = <id>;

POST /events
  └─ INSERT INTO events (...) VALUES (...);

GET /closures/<event_id>
  └─ SELECT * FROM closures_active WHERE event_id = <event_id>;

GET /congestion_zones
  └─ SELECT * FROM congestion_zones_active;

POST /route (A* Algorithm)
  ├─ SELECT * FROM events_active (untuk waypoints)
  ├─ SELECT edges FROM closures_active (untuk avoid roads)
  └─ SELECT edges, level FROM congestion_zones_active (untuk penalty costs)

GET /parking/<event_id>
  └─ SELECT * FROM parking_spots WHERE event_id = <event_id>;

GET /info/sejarah
  └─ SELECT * FROM sejarah WHERE is_active = true;

GET /info/tentang
  └─ SELECT * FROM tentang WHERE is_active = true;

12. PENTING - NOTES
================================================================================

⚠️ EDGE FORMAT (Critical untuk A* algorithm):
   ✓ Correct:  [{"from": "node_001", "to": "node_002"}]
   ✗ Wrong:    [{"source": "node_001", "target": "node_002"}]
   ✗ Wrong:    ["node_001-node_002"]
   
   Node IDs harus match dengan node IDs di road network graph Anda!
